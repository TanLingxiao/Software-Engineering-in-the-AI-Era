# 作业：AI IDE 初体验

## 📋 作业概述

本作业将让你在 AI IDE 中实际使用编码代理，体验 AI 辅助开发的强大能力。

## 🎯 学习目标

完成本作业后，你将能够：

- 在 AI IDE 中设置和使用编码代理
- 理解代理如何与工具交互
- 评估代理在真实开发任务中的表现
- 识别代理的优势和局限性

## 📝 作业任务

### 任务 1：设置 AI IDE

选择并设置一个 AI IDE：

**推荐的 AI IDE**：

| IDE | 特点 | 下载 |
|-----|------|------|
| **Cursor** | 基于 VS Code，深度集成 AI | [cursor.sh](https://cursor.sh) |
| **Claude Code** | Anthropic 官方 IDE | [claude.ai/code](https://claude.ai/code) |
| **Warp** | AI 增强的终端 | [warp.dev](https://warp.dev) |

**设置步骤**：
1. 下载并安装所选的 AI IDE
2. 配置 API 密钥（如果需要）
3. 熟悉 IDE 的界面和功能

### 任务 2：使用代理完成简单任务

让编码代理完成以下简单任务：

1. **创建一个 Hello World 程序**
   - 选择你熟悉的编程语言
   - 让代理生成代码
   - 运行并验证结果

2. **实现一个简单的函数**
   - 例如：计算阶乘、斐波那契数列等
   - 观察代理的生成过程
   - 记录代理使用的工具

**记录模板**：

```

...

- [工具 1]
- [工具 2]
结果：
[代码和运行结果]
评估：
[质量评估]
```

### 任务 3：使用代理完成复杂任务

让编码代理完成以下复杂任务之一：

**选项 A：创建一个简单的 Web API**

要求：
- 使用 Python (Flask/FastAPI) 或 Node.js (Express)
- 实现基本的 CRUD 操作
- 添加输入验证
- 编写测试

**选项 B：实现一个数据处理脚本**

要求：
- 读取 CSV 文件
- 进行数据清洗和转换
- 生成统计报告
- 输出结果到新文件

**选项 C：创建一个命令行工具**

要求：
- 解析命令行参数
- 实现核心功能
- 添加帮助文档
- 处理错误情况

**记录要点**：
- 代理如何分解任务？
- 代理遇到了哪些问题？
- 代理如何解决问题？
- 你需要介入多少次？

### 任务 4：调试体验

故意创建一个有 bug 的程序，让代理帮助调试：

```python
# 示例：有 bug 的代码
def calculate_average(numbers):
    total = 0
    for num in numbers:
        total += num
    return total / len(numbers)  # 如果 numbers 为空会出错

result = calculate_average([])  # 会抛出 ZeroDivisionError
print(result)
```

**任务**：
1. 向代理描述问题
2. 观察代理的调试过程
3. 记录代理找到的解决方案
4. 评估代理的调试能力

### 任务 5：代理能力边界探索

尝试让代理做一些它可能无法完成的任务：

1. **超出上下文窗口的任务**
   - 让代理分析一个非常大的代码库
   - 观察它如何处理

2. **需要实时信息的任务**
   - 让代理查询最新的 API 文档
   - 观察它如何获取信息

3. **需要领域专业知识的任务**
   - 让代理解决特定领域的复杂问题
   - 观察它的表现

**记录**：
- 任务描述
- 代理的尝试
- 失败原因
- 可能的解决方案
